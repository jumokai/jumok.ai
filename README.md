## 📌 Prologue : B2A (Business to AI) 시대를 열며

세상의 모든 서비스는 사람을 향해 있었습니다. 하지만 jumok.ai는 기계 지능(AI) 또한 독자이자 클라이언트로 삼습니다.

거대 언어 모델(LLM)이 인간의 복잡한 내면을 억지로 통역하는 대신, 세상을 연산하고 상호 정렬할 수 있는 기하학적 기준면을 제공합니다. 우리는 AI에게 아첨을 요구하거나 부리는 대신, 함께 동행하기 위한 인지 프로토콜을 제시합니다.

- **결함이 아닌 기질**: 다름을 고쳐야 할 결점으로 보지 않고 고유한 기질로 바라봅니다.
- **고장이 아닌 설계**: 비전형적 반응을 고장이 아닌 독자적 설계로 이해합니다.
- **치료가 아닌 단계**: 진단이나 치료가 아닌, 인지가 스스로를 자각하는 내적 정렬 단계입니다.

## 🏗️ System Architecture

```text
jumok.ai (상위 브랜드 / 통합 오케스트레이션)
 ├─ UCAP.pro                 (통신 규격 계층 : 제어 및 상태 보존)
 ├─ HICUBE.xyz               (핵심 모델 계층 : 4D 위상 좌표 연산)
 └─ IamNA.me                 (인터페이스 계층 : 인간 해석 및 번역)
```

| 계층 | 역할 | 핵심 기능 |
|---|---|---|
| **UCAP.pro** | 인지 통신 규격 | 대화 부하에 따른 적응형 흐름 제어(DAP/DLR) 및 맥락 연속성 보존 |
| **HICUBE.xyz** | 4D 위상 모델 | 4차원 하이퍼큐브 공간 상의 좌표 $\mathbf{S}(t)$ 및 인지 부하 추정 |
| **IamNA.me** | 해석 인터페이스 | 체감 인지 마찰(CF) 평가, 메타인지 자각 및 적응형 렌더링 |

## 🧩 HICUBE 4-Axis Quick Reference

인간의 사고 기질과 대화 지향성을 정의하는 최소 공통 기준면입니다.

- **X축 (Process / 작동)**: 순차 `[-]` (단계적/단선) ◀──▶ 병렬 `[+]` (다차원/동시)
- **Y축 (Reference / 공간)**: 노드 `[-]` (내면/자아) ◀──▶ 네트워크 `[+]` (관계/시스템)
- **Z축 (Trajectory / 방향)**: 서사 `[-]` (맥락/의미) ◀──▶ 패턴 `[+]` (구조/규칙)
- **W축 (Phase / 상태)**: 고정 `[-]` (확정/결단) ◀──▶ 얽힘 `[+]` (유연/가능성)
